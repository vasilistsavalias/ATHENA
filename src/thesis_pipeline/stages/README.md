# Legacy Stages

This package contains the original numbered stage implementations.

Use it for:

- implementation-level debugging of a historical stage
- understanding old stage numbering such as `02b`, `03b`, or `10b`
- compatibility with older tests and reports

Do not treat this package as the canonical stage order. The active execution model lives in:

- `thesis_pipeline.pipeline.registry`
- `thesis_pipeline.pipeline.stage_groups`
